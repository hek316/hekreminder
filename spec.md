# PRD: HekReminder — Apple Reminder Web Version

> 작성일: 2026-04-07  
> 상태: 초안 (리뷰 대기)

---

## 1. 개요

Apple Reminder App의 핵심 UX를 웹 브라우저에서 그대로 구현한다.
사용자는 리마인더를 목록(List)으로 관리하고, 각 항목에 마감일·우선순위·메모를 부여할 수 있다.

### 목표
- Apple Reminder의 주요 기능을 웹에서 사용 가능하게 한다
- 깔끔한 macOS/iOS 스타일 UI를 구현한다
- 빠른 입력 경험(키보드 중심)을 제공한다

## 3. 핵심 기능 (MVP Scope)

### 3.1 스마트 목록 (읽기 전용 필터)

Apple Reminder의 상단 4개 카드를 동일하게 구현한다.

| 스마트 목록 | 조건 |
|------------|------|
| 오늘 | dueDate가 오늘 날짜이고 완료되지 않은 항목 |
| 예정 | dueDate가 미래이고 완료되지 않은 항목 |
| 전체 | 완료되지 않은 모든 항목 |
| 완료됨 | completed = true인 항목 |

### 3.2 사용자 정의 목록 (List)

- 목록 생성 / 이름 수정 / 삭제
- 목록별 색상 지정 (Apple의 색상 팔레트 기반)
- 목록별 아이콘 지정 (SF Symbols 대응 이모지)
- 목록 내 리마인더 수 표시

### 3.3 리마인더 (Reminder)

각 리마인더는 특정 목록에 속한다.

| 필드 | 설명 |
|------|------|
| title | 리마인더 제목 (필수) |
| notes | 메모 (선택) |
| dueDate | 마감 날짜 (선택) |
| dueTime | 마감 시각 (선택, dueDate 있을 때만) |
| priority | 없음 / 낮음 / 중간 / 높음 |
| completed | 완료 여부 |
| flagged | 깃발 표시 여부 |

**주요 동작:**
- 체크박스 클릭으로 완료 토글 (완료 시 취소선 애니메이션)
- 인라인 편집 (클릭 즉시 편집 가능)
- 엔터로 다음 리마인더 생성 (Apple과 동일한 UX)
- 삭제는 스와이프 또는 우클릭 컨텍스트 메뉴

### 3.4 서브태스크 (Subtask)

- 각 리마인더 하위에 서브태스크 추가 가능
- 서브태스크도 완료 토글 지원
- 서브태스크가 있는 경우 리마인더 우측에 진행률 표시 (예: 2/5)

### 3.5 깃발 목록 (Flagged)

- 상단 스마트 목록에 "깃발"을 추가 (flagged = true인 항목 모음)

## 4. 데이터 모델 변경 계획

현재 `Reminder` 엔티티에서 아래 필드를 추가/변경한다.

```
Reminder (기존 + 추가)
├── id
├── title
├── notes          (description → notes로 rename)
├── dueDate
├── dueTime        [추가]
├── priority       [추가] NONE | LOW | MEDIUM | HIGH
├── completed
├── flagged        [추가]
├── list           [추가] → ReminderList (N:1)
└── subtasks       [추가] → List<Subtask> (1:N)

ReminderList [새 엔티티]
├── id
├── name
├── color          (HEX 문자열)
├── icon           (이모지 또는 식별자)
└── reminders      (1:N)

Subtask [새 엔티티]
├── id
├── title
├── completed
└── reminder       (N:1)
```

## 5. API 설계

### 5.1 목록 API

| Method | Path | 설명 |
|--------|------|------|
| GET | /api/lists | 전체 목록 조회 (리마인더 수 포함) |
| POST | /api/lists | 목록 생성 |
| PUT | /api/lists/{id} | 목록 수정 |
| DELETE | /api/lists/{id} | 목록 삭제 (하위 리마인더 포함) |

### 5.2 리마인더 API

| Method | Path | 설명 |
|--------|------|------|
| GET | /api/reminders | 전체 조회 (쿼리파라미터: listId, filter=today/scheduled/flagged/completed) |
| GET | /api/reminders/{id} | 단건 조회 |
| POST | /api/reminders | 생성 |
| PUT | /api/reminders/{id} | 수정 |
| PATCH | /api/reminders/{id}/complete | 완료 토글 |
| PATCH | /api/reminders/{id}/flag | 깃발 토글 |
| DELETE | /api/reminders/{id} | 삭제 |

### 5.3 서브태스크 API

| Method | Path | 설명 |
|--------|------|------|
| POST | /api/reminders/{id}/subtasks | 서브태스크 생성 |
| PUT | /api/reminders/{id}/subtasks/{subId} | 서브태스크 수정 |
| PATCH | /api/reminders/{id}/subtasks/{subId}/complete | 완료 토글 |
| DELETE | /api/reminders/{id}/subtasks/{subId} | 삭제 |

### 6.2 사이드바 상세 스펙

#### 스마트 목록 카드 (상단 2×2 그리드)

Apple Reminder의 상단 카드 4개를 동일하게 구현한다.

| 카드 | 아이콘 | 배경색 | 표시 숫자 |
|------|--------|--------|-----------|
| 오늘 | 📅 (calendar) | `#007AFF` (파랑) | 오늘 마감 미완료 수 |
| 예정 | 📅 (calendar with clock) | `#FF3B30` (빨강) | 미래 마감 미완료 수 |
| 전체 | ≡ (lines) | `#000000` (검정) | 전체 미완료 수 |
| 깃발 | 🚩 | `#FF9500` (주황) | 깃발 표시 미완료 수 |

- 카드 크기: `calc(50% - 4px)` × 약 80px (정사각형에 가깝게)
- 카드 내부: 좌측 하단에 레이블, 우측 상단에 흰색 숫자 (큰 폰트)
- 아이콘: 카드 배경색과 동일 계열의 SF Symbol 스타일 아이콘
- 카드 radius: `12px`
- 호버 시: 약간 밝아지는 효과 (`brightness(1.08)`)
- 선택 시: 눌린 효과 (`scale(0.97)`)

#### 나의 목록 섹션

```
나의 목록
──────────────────────
● 개인                5
● 업무                8
● 건강                2
──────────────────────
+ 목록 추가
```

- 섹션 헤더: `나의 목록` — 11px, 회색(`#8E8E93`), 대문자 없음
- 목록 행: `40px` 높이
  - 왼쪽: 색상 원형 아이콘 (`28px`, 목록 색상으로 채워진 원) + 이름
  - 오른쪽: 미완료 리마인더 수 (회색 숫자)
- 선택된 목록: 행 전체 `#E5E5EA` 배경, `border-radius: 8px`
- `+ 목록 추가`: 클릭 시 인라인 입력 필드 노출 (이름 즉시 입력)

#### 목록 색상 팔레트 (Apple 동일)
`#FF3B30` `#FF9500` `#FFCC00` `#34C759` `#00C7BE` `#30B0C7` `#007AFF` `#5856D6` `#AF52DE` `#FF2D55` `#A2845E` `#8E8E93`

### 6.3 메인 컨텐츠 영역 상세 스펙

#### 헤더

```
개인                              편집
```

- 제목: 목록 색상과 동일한 색상, `bold`, `28px`
- 우측: `편집` 버튼 (클릭 시 다중 선택/삭제 모드)
- 스마트 목록 선택 시: 해당 카드 색상으로 제목 표시

#### 리마인더 행

```
○  마트 다녀오기                    🚩  📅 오늘
   우유, 계란, 빵 사기
```

- 체크박스: `20px` 원형, 기본은 목록 색상의 `2px` 테두리만 있는 빈 원
  - 완료 시: 목록 색상으로 꽉 채워지고 흰색 체크 표시 (300ms 애니메이션)
  - 완료 처리된 행: 제목에 취소선, 회색으로 페이드 (500ms)
- 행 높이: 메모 없으면 `44px`, 메모 있으면 가변
- 메모(notes): 제목 아래 `13px`, `#8E8E93` 색상, 한 줄 truncate
- 우선순위 표시 (제목 왼쪽, 체크박스 오른쪽):
  - 높음: `!!!` 빨간색
  - 중간: `!!` 주황색
  - 낮음: `!` 파란색
- 마감일 표시 (우측):
  - 날짜만 있을 때: `오늘`, `내일`, `목`, `3월 5일` 형식
  - 시각 있을 때: `오전 9:00` 추가
  - 기한 초과: 빨간색 텍스트
  - 깃발: 🚩 오렌지색 아이콘 (날짜 오른쪽)
- 서브태스크 있는 경우: 제목 우측에 `2/5` (완료/전체) 작은 회색 텍스트

#### 서브태스크 행

```
   ○  서브태스크 제목            (18px 들여쓰기)
```

- 부모 리마인더와 동일한 스타일, 왼쪽 `32px` 들여쓰기
- 펼침/접힘 없이 항상 노출 (Apple과 동일)

#### 완료된 항목 섹션

```
▶ 완료됨   (3)
```

- 목록 하단에 접힌 상태로 표시
- 클릭 시 펼쳐져 완료 항목 표시

#### 새로운 리마인더 추가

```
+ 새로운 리마인더
```

- 목록 최하단 고정
- 클릭 시 빈 입력 행 즉시 생성 (focus 이동)
- `Enter` → 저장 후 다음 빈 입력 행 생성
- `Escape` → 취소
- `Tab` → 서브태스크 전환

### 6.4 상세 편집 패널 (우측 슬라이드)

리마인더 클릭 시 우측에서 슬라이드 인 (320px 너비)

```
────────────────────────
  세부 사항          ✕
────────────────────────
  제목
  ┌────────────────────┐
  │ 마트 다녀오기       │
  └────────────────────┘

  메모
  ┌────────────────────┐
  │ 우유, 계란, 빵     │
  └────────────────────┘

  📅 날짜          오늘 ▶
  ⏰ 시각    오전 9:00 ▶
  🚩 깃발              ○
  ! 우선순위         없음 ▶

  서브태스크
  ○ 1장 읽기
  ○ 메모 작성
  + 서브태스크 추가
────────────────────────
```

- 각 항목은 행(row) 스타일, 클릭 시 picker/input 노출
- 날짜 picker: 캘린더 드롭다운 (macOS 스타일)
- 시각 picker: 시/분 드럼롤 또는 input[type=time]
- 우선순위: `없음 / ! / !! / !!!` 세그먼트 선택
- 패널 외부 클릭 시 자동 저장 후 닫힘
